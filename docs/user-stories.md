1. Получить курс на дату.
Как бухгалтер я хочу получать актуальный курс выбранной валюты на указанную дату, чтобы верно расчитывать рыночную стоимость зарубежных активов.
Критерии приемки:
a. Необходимую валюту нужно указывать в request (доллар, евро, юань, теньге).
b. Базовая валюта по умолчанию - рубль.
c. Запрос по API возвращает курс и дату актуальности.
d. Ошибки внешних источников обрабатываются и логируются.

2. История изменений курса.
Как финансовый аналитик я хочу видеть историю изменения курса валюты, чтобы верно спрогнозировать дальнейшую волатильность.
Критерии приемки:
a. В базе хранится курс, даат курса и источник.
b. Возможен запрос истории по валюте и периоду.
c. Максимальный период выгрузки 365 дней.
d. Результат упорядочен по дате (по умолчанию по убыванию)

3. Список курсов.
Как платежный сервис я хочу получать список из нескольких курсов валют одним запросом, чтобы быстрее получать и обрабатывать данные.
Критерии приемки:
a. Через один запрос по API можно получить список разных курсов на дату.
b. В запросе можно передать несколько валют (доллар, евро, юань, теньге).
c. API возвращает список курсов в одном формате.
d. Если одна из валют недоступна - сервис не падает, а возвращает статус по каждой валюте.

4. Курс с разных платформ.
Как сервис конвертации я хочу получать курс валюты с разных источников, чтобы выбирать наилучший курс для расчетов.
Критерии приемки:
5. Список доступных источников - ЦБ РФ, Мос Биржа, СПБ биржа.
6. Запрос по API возвращает курс валюты по каждому доступному источнику.
7. В ответе указывается источник и время обновления.
8. Если источник недоступен - сервис возвращает статус unavailable по нему.
